Functional Programming in Scala Capstone Project

This scala capstone project is the last course of the Functional Programming in Scala Specialization Series from Coursera. This writing is about my implementation solution of the project, with a little bit of mathmatical model and performance measurements of the project. This project is composed of 6 milestones to implement a complete application which processes a lot of weather data, and then shows interactive visualization of the weather distribution around the world and its evolution over time on the web, like image below.

This image was captured from the interactive web page as my project implementation result. The implementaion code of each project milestone will be posted in my github, if it’s OK with the Coursera’s policy.

The development of the application in this project involves transforming data collected in weather stations around the world into meaningful information such as the average temperature of each year. It then produces world temperature images using spatial and linear interpolation techniques and finally, implements the user interface that reacts to the user’s actions.

This application is very CPU intensive in that it uses CPU intensive mathmatical model to build a lot of image tiles of weather map around the world. My initial implementation of image creation part took more than 2 hours to compute every tiles for year 2015 alone with my laptop computer.

In the capstone project, temperature data measured from the year 1975 to 2015 were given in csv files. The application reads these csv data files and then based on this data, it mathmatically predicts temperature values of each pixel in a tile(256x256 pixels) to produce world temperature images like picture above.

Milestone 1 through 5 of this project are to generate tiles of temperature images for zoom level 0, 1, 2 and 3 for each year from 1975 to 2015. Level 0 generates 1 tile, level 1 generates 4 tiles, level 2 generates 16 tiles, and level 3 generates 64 tiles, to cover the global tmeperature images. The last milestone is to implement interactive web using Reactive Signal and ScalaJS.

Predicting temperatures of every pixel point around the world is based on the inverse distance weighting algorithm using sample temperature data in csv files.

Mathmatically, predicted temperature at any location \(\mathbf x\) on the surface of the Earth, denoted by \(u(\mathbf x)\), is

$$u(\mathbf x) = {\sum_{i=1}^N w_i(\mathbf x)u_i \over \sum_i^N w_i(\mathbf x)}$$

where $$w_i(\mathbf x) = {1 \over d(\mathbf x, \mathbf x_i)^p}$$

and \(N\) is total number of sample temperatures given in csv files.

Here, \(p \ge 2.0\)(more about this value later), and the distance of the shortest path between point \(\mathbf x\) and point \(\mathbf x_i\) on the surface of the Earth

$$d(\mathbf x, \mathbf x_i) = r\Delta\sigma$$ where Earth Radius \(r = 6371km\), and $$\Delta\sigma = arccos(sin\phi_1sin\phi_2 + cos\phi_1cos\phi_2cos(\Delta\lambda))$$

where \(\phi_1, \lambda_1\) and \(\phi_2, \lambda_2\) are the geographical latitude and longitude of two points \(\mathbf x\) and \(\mathbf x_i\), illustrated in the image below as P and Q.

Refer to here for more information on Great-circle distance.

In case \(d(\mathbf x, \mathbf x_i) \le 1km\), then \(u(\mathbf x) = u_i\) which means that predicted temperature value of the location near measured location is approximately the measured temperature value.

Again, \(u(\mathbf x)\) is predicted temperature at location \(\mathbf x\) with latitude \(\phi\) and longitude \(\lambda\).

Processing mathmatical formula like this in Scala is not that difficult, but generating a lot of images using this formula is very CPU intensive and takes long time.

Also, to pass the grader of the project, the implementation of APIs in each milestone must perform fast, utilizing all of your CPU cores in parallel and memory space optimally, as the grader restricts run-time and memory space.

The project comes with SBT build settings that include Spark, Akka Streams, Monix and fs2 as parallel and distributed computing tools. But they are optional. I used Spark for data join and other operations against temperature data at first, but Spark is actually distributed parallel computation framework.

Since I have only one laptop, using Spark didn’t make sense for gaining performance as managing spark context overhead in a single node costs more than gaining performance over spark parallelism.
I could have deployed Spark cluster on AWS EC2 nodes and run my application on it. But fortunately, using scala standard parallel collection library was enough to pass the grader successfully to get full score of 10⁄10.

The mathmatical formula above to compute \(u(\mathbf x)\) allows you to do data parallel map computation against the collection of sample temperatues. Once the parallel map computation is done, you can then sum the result collection up to calculate \(u(\mathbf x)\).

In fact, there are other places in implementation code appropriate to parallelize computing and this is one of them. But it should be noted that parallelism should be applied only when the size of data exceeds certain threshold, because applying too many parallelism can cause computing overhead, hence performance degradation.

Once each milestone API is implemented, they are used to generate the tiles of images and check performance of the solution in the Main object.

object Main extends App {
  import Extraction._
  import Visualization._
  import Interaction._

  def time[R](block: => R): (R, String) = {
    val start = System.currentTimeMillis()
    val result = block
    val end = System.currentTimeMillis()
    (result, s"${end - start}")
  }

  val colorScale = Seq((60d, Color(255, 255, 255)), (32d, Color(255, 0, 0)), 
    (12d, Color(255, 255, 0)), (0d, Color(0, 255, 255)),(-15d, Color(0, 0, 255)), 
    (-27d, Color(255, 0, 255)), (-50d, Color(33, 0, 107)), (-60d, Color(0, 0, 0)))

  //callback to create tiles of world temperature image
  def callmeBack(year: Int, zoom: Int, x: Int, y: Int, 
                 data: Iterable[(Location, Double)]): Unit = {
    val (image, timeTaken) = time(tile(data, colorScale, zoom, x, y))
    val path = image.output(s"target/temperatures/${year}/${zoom}/${x}-${y}.png")
    println(s"$path: $timeTaken milliseconds taken")
  }

  val yearlyData = (2015 to 1975 by -1).map { y =>
    (y, locationYearlyAverageRecords(locateTemperatures(y, "/stations.csv", s"/${y}.csv")))
  }

  generateTiles(yearlyData, callmeBack)
}

It took more than 3 hours generating images of year 2015 alone. It would take a couple of days to generate all the images from 1975 to 2015, utilizing all 4 cores of my laptop computer.

I was afraid that it could heat uplimit and crash my laptop over the course of such a CPU intensive, long running computation. So I interrupted the running process in the middle.

Fortunately, there was a solution for this. Instead of using the inverse distance weighting algorithm for predicting temperatures, there is faster algorithm called bilinear interpolation, at the cost of a little bit less quality of image.

It first predicts 180(lat) by 360(lon) grid temperatures of the Earth surface using the inverse distance weighting algorithm, and then once you have this grid cached during the creation of the first image tile, predicting temperatures using bilinear interpolation algorithm is very fast for the creation of the subsequent tiles.

The algorithm was applied with unit square bilinear interpolation because each square in the grid is considered unit square.

Mathmatically, given a coordinate system in which the four corners of a unit square are (0, 0), (0, 1), (1, 0), and (1, 1), then the interpolation formula is:

\(f(x, y) \approx \begin{bmatrix}1-x & x\end{bmatrix} \begin{bmatrix}f(0,0) & f(0,1) \\ f(1,0) & f(1,1) \end{bmatrix} \begin{bmatrix}1-y & y \end{bmatrix}\)

where, \(f(x,y)\) is bilinearly interpolated temperature at longitude x and latitude y, and f(0, 0) through f(1, 1) are precomputed, or cached temperatures of 4 corners of a unit square in the grid.

Coordinates are shown in the picture below, as illustrated in wikipedia.
Consider top left corner \((X_1,Y_2)\), or \(Q_{12}\) as (0, 0) and bottom right corner \((X_2, Y_1)\), or \(Q_{21}\) as (1, 1) in this project.
P corresponds to \(f(x,y)\).

Source from wikipedia

For those who are confused how to apply this algorithm, here is an example. Let’s say predicting temperature of a location with longitude 128.254 and latitude 36.923. Then, the coordinate (0,0) maps to (128, 37), and (1, 0) maps to (129, 37), (0, 1) maps to (128, 36), and (1, 1) maps to (129, 36). With this at hand, you can apply unit square bilinear interpolation formula.

Once you implement bilinear interpolation APIs in Visualizaion2 module, you may rewrite driver program like this:

import Visualization2._
...

//generate image fast using bilinear interpolation algorithm
def callmeBackGrid(year: Int, zoom: Int, x: Int, y: Int, 
                   data: Iterable[(Location, Double)]): Unit = {
  val (image, timeTaken) = time(visualizeGrid(makeGrid(data), colorScale, zoom, x, y))
  val path = image.output(s"target/temperatures/${year}/${zoom}/${x}-${y}.png")
  println(s"${path}: ${timeTaken} milliseconds taken")
}

val yearlyData = (1975 to 2014).map { y =>
  (y, locationYearlyAverageRecords(locateTemperatures(y, "/stations.csv", s"/${y}.csv"))) }


generateTiles(yearlyData, callmeBackGrid)

This is huge speed up.

Note that it takes long time to build the first image(1975\0\0-0.png) as it needs to cache up grid data while generating the image. Once it caches grid, building subsequent images need no time as they are built based on the already cached grid when applying bilinear interpolation algorithm. It produced all images from 1975 to 2014 no longer 2 hours with my laptop!

One thing to note: the value \(p\) in \(w_i(\mathbf x) = {1 \over d(\mathbf x, \mathbf x_i)^p}\)

I tried from 2.0, 3.0, 4.0, 4.5, 5.0 for the value p, and I found value 4.5 is best performant. I posted the result of this project here and if you look at the temperature image of year 2013(using the slide bar handle), you will see image with p value 2.0. I generated the image with p value 2.0 just for example purpose. All other images are created with p value 4.5

There are other images that need to be generated in this porject. They are deviation images, which look something like below.

Deviation images are images of recent temperature changes from average temperature in the past. The average temperature is temperature averaging from year 1975 to 1989. This average temperature is considered normal and the deviation images show how much the tempertures have changed over each year from 1990 to 2015, compared to normal temperature.

These images are again generated using bilinear interpolation algorithm for speed up. Assuming that you implemented the Manipulation module, the implementation code generating these images is like below:

  import Manipulation._
  ...

  //generate temperature deviation images 
  val colorScale2 = Seq((7d, Color(0, 0, 0)), (4d, Color(255, 0, 0)), (2d, Color(255, 255, 0)), 
    (0d, Color(255, 255, 255)), (-2d, Color(0, 255, 255)), (-7d, Color(0, 0, 255)))

  val temperaturess = (1975 to 1989).map{ y => 
    locationYearlyAverageRecords(locateTemperatures(y, "/stations.csv", s"/${y}.csv")) }

  val normals: (Int, Int) => Double = average(temperaturess)

  def callmeBackDeviation(year: Int, zoom: Int, x: Int, y: Int, 
                          data: Iterable[(Location, Double)]): Unit = {
    val (image, timeTaken) = time(visualizeGrid(deviation(data, normals), colorScale2, zoom, x, y))
    val path = image.output(s"target/deviations/${year}/${zoom}/${x}-${y}.png")
    println(s"$path: $timeTaken milliseconds taken")
  }

  val yearlyData = (1990 to 2015).map { y =>
    (y, locationYearlyAverageRecords(locateTemperatures(y, "/stations.csv", s"/${y}.csv"))) }

  generateTiles(yearlyData, callmeBackDeviation)

Final milestone is to implement an interactive user interface using Signal abstraction and Scala.js, which seem to be very useful tools for developing reactive web application. You will need to know how Signal works, so read Signal source code in the project carefully to understand how signals propagate data around.

It is good to understand basics on how Scala.js works, but you really don’t need to know about it to make it happen in this project. You just follow the instructions in the course and you will get idea.

Once all the milestones are successfully passed by the grader, then you are ready to observe the global climate changes on the interactive web. Just follow the instruction in the final milestone and you will have an interactive image web, like my result here.
